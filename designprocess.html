Milestone 2 Goals

  • Day 1: Create classes for “Rob”, “Guards”, “Cameras” and all characters in the game.
  • Day 2: Create the map for the first level (have a general idea and put partly into action).
  • Day 3: Finish map, finish characters, make sure everything works as intended (moving). Collision detection with doors.
  • Day 4: Implement “lockpick” mechanism, Make the win condition when “Rob” robs a certain item (most likely a safe) and returns to the starting point.
  • Day 5: Test Run and Fixes. If it works, Extra Credit.

Milestone 3 Goals

  • Day 1: Bug Fixes, ensure Level 1 is complete.
  • Day 2: Create Level 2, add new cameras and doors. Character Descriptions?
  • Day 3: Create Level 3, add new cameras and doors.
  • Day 4: UI and Interactions.
  • Day 5: Final Test Run and Fixes. If it works, Extra Credit?

Roles/Responsibilities

  • Arjun: Character Design
  • Emerson: Overall Level Mechanisms
  • Philip: Different abilities/unique attributes

Our game targets all three Accessibility focus areas by having different levels of difficulty for different players, being open minded to change the game based on player feedback, and lastly every level will have guidelines, and everytime a player dies they will get a hint.
